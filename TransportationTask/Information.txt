
   Программа решает транспортную задачу 
с  помощью  метода потенциалов,
опорный план может  быть найден методом северо-западного угла
или методом  минимального  элемента.   
   
  Для чтения задачи  из файла задача должна быть 
записана в файле  в таком формате,

  числа  записываются  в  файле  без запятых и
 других знаков  препинания, 
 отделяются любым количеством  пробелов,

 n  m

   A1 A2 ...  An
   B1 B2  ... Bm

 a11   a12    ...  a1m
 a21   a22    ...  a2m
   ................
 an1 an2      ... anm

 n - количество поставщиков,
 m - количество  потребителей,
 A - вектор  количеств  товаров  у  поставщиков,
 B - вектор потребностей  в  товарах  у  потребителей,
 a - матрица  стоимостей,

       Размер формы  можно менять и размер таблиц
   для  вкладки ход решения можно менять  с  помощью  мышки,
      для  матрицы E  выводятся значения -cij
      -cij=aij-ui-vj 
	 в матрице потенциалов записаны значения  
       cij=ui+vj-aij , в оптимальном плане все такие значения
   элементов матрицы потенциалов должны
   быть отрицательны или равны нулю,
        на основе матрицы потенциалов 
   построены вычисления в программе,
       на форме при этом отображаются элементы матрицы E,
   которые в оптимальном плане  должны быть 
   положительны  или равны нулю.
	  

    Действия при выборе пунктов меню
1) Задача из файла - прочитать и решить задачу из файла,
2) Заполнить - создать задачу на основе случайных чисел
               и решить ее,
3) Решить - решить задачу, которая набрана на форме,
4) Очистить - очистить данные по задаче и решению на форме,
5) Сохранить задачу - сохранение условия последней 
решенной задачи, если не был выбран пункт меню "Очистить",

       Для  пункта  меню "Решить" 
перед  чтением задачи  редактирование ячейки должно 
быть закончено,  то есть должно отображаться 
значка ручки в заголовке строки,

      Для  пункта  меню "Сохранить  задачу",
сохраняется закрытая задача, которая была решена на форме,
то есть сохраняется  задача  уже  приведенная 
к виду  закрытой задачи,

     каждый  шаг  решения  соответствует 
  одному  плану  решения.
   
     После решения задачи  можно  выбрать  другой
метод   построения  опорного  плана  и  решение задачи 
автоматически  выполнится  с новым  методом выбора 
опорного плана,
     если  выбрать  пункт "Любой  метод",  то 
метод  построения  опорного  плана  будет  выбиратся 
с помощью  генерации случайных  чисел, 
будет выбран  один  из двух методов,
метод северно-западного  угла  или метод  минимального 
элемента. 
 

   Задача  может  быть
1)  прочитана из файла,
2)  сформирована из случайных чисел,
3)  непосредственно набрана на форме.

    Путь для чтения и сохранения файлов 
\TransportationTask\TransportationTask\bin\Debug
в этой папке есть 4 файла с примерами из учебника и лекций,
   1.txt - пример из учебника Кузнецова, страница 110,
   2.txt , 3.txt, 4.txt - примеры из лекций.

    Для  формирования задачи  на основе случайных чисел 
нужно  сначала  задать на форме количество поставщиков
и количество  потребителей.
    
	Если задача была решена, то ее условие можно сохранить
в файл,  чтобы прочитать при следующей работе с программой,
сохраняется соответствующая  закрытая задача  
только  для  той задачи, для которой было выполнено
решение.
